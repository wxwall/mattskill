01HyperFrames — HTML/GSAP 动效引擎
OpenMontage 有两套合成引擎：Remotion（React 组件写视频）和 HyperFrames（HTML + CSS + GSAP 动画）。HyperFrames 适合做网页级动效：
✨
SplitText 逐词动画
文字逐词/逐字母弹出，TikTok/Reels 风格的快节奏字幕效果
📊
数据组件注册表
hyperframes add data-chart / shader-transition —— 即插即用的视觉组件
🌐
网站转视频
输入一个 URL → 自动提取配色/字体 → 生成品牌一致的社会化广告
# HyperFrames 需要 Node.js ≥ 22
npx @hyperframes/cli

02质量门 — 确保产出靠谱
OpenMontage 在多个环节设置了自动质量检查，不需要你手动审查：
🔍
Pre-compose 验证
合成前检查——素材齐全？字幕长度匹配旁白？分辨率正确？
📊
幻灯片风险评分
自动检测"只是图片+文字"的幻灯片式视频，提示添加动效。评分低于阈值 → 建议改进。
🔧
Post-render 自审
渲染后用 ffprobe 检查分辨率/码率，帧采样确认画面质量，音频分析检测爆音/静音段。
💰
预算治理
执行前估算成本、可设消费上限、可配置单次操作的审批阈值。
03Provider 评分选择机制
当你配了多个同类型 API（比如 Veo + Kling + Runway），Agent 不会随机选——它用 7 维度评分来决定用哪个：
任务匹配度输出画质控制精度可靠性成本延迟风格连续性
你不需要手动调这个——Agent 自动完成评分和选择。但了解这个机制有助于理解：为什么有时候 Agent 选了 Veo 而不是 Kling？可能是画质和控制精度需求不同。
04自定义流水线
OpenMontage 的 12 条流水线定义在 pipeline_defs/目录下，都是 YAML 文件。你可以：
📝
复制现有流水线改参数
比如把"animated_explainer"的默认图表从 3 个改成 5 个
🔧
写新的 skill 文件
在 skills/pipelines/ 下创建 Markdown，定义新流程的 Stage Director 指令
🎨
创建视觉风格 playbook
在 styles/ 下写 YAML，定义配色、字体、转场风格。已有 Clean Professional、Flat Motion Graphics 等模板。
05学习路径总结

HyperFrames + 自定义流水线 + 风格 playbook。
