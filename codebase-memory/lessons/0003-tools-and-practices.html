0114 个 MCP 工具一览
📂 索引管理（4 个）
index_repository — 索引整个仓库。codebase-memory-mcp 的入口命令，扫描项目并构建知识图谱。
list_projects — 列出所有已索引的项目及其状态。
delete_project — 删除指定项目的索引数据。
index_status — 查看索引状态：文件数、符号数、节点数、边数。
🔍 图查询（4 个）
search_graph — 按名称或模式搜索图中的符号。最常用的查询工具。
trace_path — 追踪两个符号之间的调用路径。CodeGraph 也有类似功能。
query_graph — 独特功能：使用 Cypher-like 查询语言直接查询图结构。可以写类似 MATCH (f:Function)-[:CALLS]->(g:Function) RETURN f, g 的查询。
get_graph_schema — 获取知识图谱的 schema，了解有哪些节点类型和边类型。
📄 代码分析（3 个）
get_code_snippet — 获取指定符号的源码片段，替代 Agent 自己读文件。
get_architecture — 独特功能：获取项目的架构概览，包括模块结构、依赖关系、路由映射。
search_code — 全文搜索代码内容。
🧠 高级功能（3 个）
detect_changes — 检测文件变更，与索引对比找出修改过的文件。
manage_adr — 独特功能：管理 Architecture Decision Records（架构决策记录）。可以在代码库中记录为什么做了某个架构决策，供 AI Agent 和团队成员查阅。
ingest_traces — 独特功能：摄入可观测性 trace 数据。将 Jaeger/Zipkin 等 APM 系统的调用 trace 关联到代码图谱，实现"线上报错→定位到代码行"。
02独特功能详解
🔮
query_graph — Cypher-like 图查询
Agent 可以直接写类 Cypher 查询来探索代码图。例如："找到所有调用了 authenticate() 的函数" 翻译成图查询，一次返回完整结果。CodeGraph 没有这个能力。
📋
manage_adr — 架构决策记录
ADR 是记录架构决策的轻量文档。codebase-memory-mcp 将 ADR 关联到知识图谱中的相关代码节点。新加入项目的开发者或 AI Agent 可以问"为什么这里用了 Redis 而不是 Memcached"，直接获得历史决策上下文。
📊
ingest_traces — Trace 关联代码
将 Jaeger/Zipkin 的分布式 trace 数据摄入到知识图谱中，关联到具体的函数调用。当线上出现慢请求时，Agent 可以直接定位到耗时函数，无需人工跳转多个 APM 页面。
03最佳实践清单
✅
先 index 再 serve
首次使用时先手动运行 index_repository 确保图构建完成，再启动 MCP serve。
✅
用 get_architecture 让 Agent 快速度项目
接手新项目时，先让 Agent 调用 get_architecture 获取项目架构概览，比读 README 更快。
✅
利用 ADR 记录关键决策
在项目里程碑处创建 ADR 记录架构决策。Agent 后续修改代码时会自动参考这些记录，避免"改了半天发现之前有人讨论过这个问题"。
✅
trace 数据关联线上问题
生产环境出现性能问题时，将 Jaeger/Zipkin trace 摄入图谱。Agent 可以直接问"最近的慢请求集中在哪些函数"。
✅
与 CodeGraph 互补使用
两个工具可以同时配置。CodeGraph 适合轻量快速索引（24 种语言），codebase-memory 适合深度图查询和 158 种语言支持。根据需要选择不同场景使用。
