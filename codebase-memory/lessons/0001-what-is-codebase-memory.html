"codebase-memory-mcp 是一个高性能代码智能 MCP Server。它用 tree-sitter 解析代码，在内存中构建知识图谱，通过 14 个 MCP 工具让 AI Agent 以亚毫秒级速度查询代码结构——单个静态二进制，零依赖。"
01与 CodeGraph 有什么不同？
如果你已经学过 CodeGraph，理解 codebase-memory-mcp 非常容易——它们解决的是同一个问题（AI Agent 读代码的效率），但实现路径不同。
1
🌐 158 种语言 vs ~24 种
CodeGraph 支持 24 种语言，codebase-memory 编译了 158 种 tree-sitter 语法——几乎覆盖所有你能想到的语言，包括 QML、CFML 等小众语言。
2
⚡ RAM-first 流水线
CodeGraph 用 SQLite 持久化存储；codebase-memory 用内存优先 + LZ4 压缩，亚毫秒级查询。Linux 内核（28M LOC、75K 文件）只需 ~3 分钟完成索引。
3
🔬 Hybrid LSP 语义解析
对 Python、TypeScript、JavaScript、PHP、C#、Go、C/C++、Java、Kotlin、Rust 做类型级语义解析，不只是 AST 层面的符号提取。
4
📦 单个静态二进制
CodeGraph 需要 Node.js 运行时；codebase-memory 是单个静态二进制（macOS/Linux/Windows），零依赖，无需 Docker、无需 Node.js、无需任何运行时。
02核心架构
codebase-memory-mcp 的工作流水线：
步骤 1
Tree-sitter 解析
158 种语言 AST 提取
↓
步骤 2
RAM-first 构建
内存中建图 + LZ4 压缩
↓
步骤 3
Hybrid LSP 解析
类型级语义解析（10 种语言）
↓
步骤 4
14 个 MCP 工具
亚毫秒级查询服务
🌳
Tree-sitter 解析
扫描源码文件，生成 AST，提取函数、类、接口、调用关系、HTTP 路由等。支持 158 种语言，所有语法编译在单个二进制中。
⚡
RAM-first 流水线
在内存中构建知识图谱，用 LZ4 做压缩。相比 SQLite 持久化方案，查询速度提升 100-1000 倍。Linux 内核 28M 行代码仅需 ~3 分钟。
🔬
Hybrid LSP 语义解析
对主流语言做类型级解析——不只知道"这个函数叫什么"，还知道它的参数类型、返回值类型、泛型约束等语义信息。
🔌
14 个 MCP 工具
通过 MCP 协议暴露给 AI Agent，涵盖索引管理、图搜索、路径追踪、变更检测、ADR 管理、Trace 摄入等。
