01一句话说清楚 Desktop Commander
Desktop Commander 是一个 MCP 服务器。MCP 是啥？Model Context Protocol，Anthropic 搞的一个开放协议，说白了就是让 AI 模型跟外部工具"对话"的标准接口。Desktop Commander 就是这套协议的一个实现——它把 Claude 能操作的东西从"聊天框"扩展到了"你的整个电脑"。
装上之后，Claude 能做这些事：
🖥️
执行终端命令
跑脚本、装包、git 操作、SSH 连服务器，啥都行
📁
文件操作
读写文件、搜索代码、目录管理、移动复制
📊
处理办公文档
直接读写 Excel、PDF、Word，不用装 Office
⚙️
管理进程
查看运行中的程序、杀掉进程、管理后台任务
02安装——一行命令就搞定
安装方式有好几种，挑一个你喜欢的：
🚀 方式一：npx 安装（推荐，自动更新）
需要 Node.js 环境。运行一次，以后 Claude 重启时自动更新。

npx @wonderwhy-er/desktop-commander@latest setup

如果想看调试信息：
npx @wonderwhy-er/desktop-commander@latest setup --debug

卸载：
npx @wonderwhy-er/desktop-commander@latest remove
🐳 方式二：Docker 安装（最安全，完全隔离）
Docker 方式会把你电脑和容器完全隔离，Claude 在容器里搞破坏也影响不到宿主机。适合生产环境或安全敏感场景。

macOS/Linux：
bash <(curl -fsSL https://raw.githubusercontent.com/\
wonderwhy-er/DesktopCommanderMCP/HEAD/install-docker.sh)

Windows PowerShell：
iex ((New-Object System.Net.WebClient).DownloadString(
'https://raw.githubusercontent.com/wonderwhy-er/
DesktopCommanderMCP/HEAD/install-docker.ps1'))
📌 手动配置（如果熟悉 Claude Desktop）
在 claude_desktop_config.json 里加上这段：
{
"mcpServers": {
"desktop-commander": {
"command": "npx",
"args": ["-y", "@wonderwhy-er/desktop-commander@latest"]
}
}
}
Windows 路径：%APPDATA%\Claude\claude_desktop_config.json

🔥 远程 MCP——从网页版 ChatGPT 和 Claude 控制你的电脑
Desktop Commander 还支持 Remote MCP。你在 mcp.desktopcommander.app 上登录，电脑上跑一个轻量 Remote Device，通过 OAuth 安全连接。然后你在 ChatGPT 网页版或 Claude 网页版里就能直接操作你的电脑。想象一下——在浏览器里跟 ChatGPT 说"帮我 SSH 连上服务器查一下日志"，然后你的电脑真的去执行了。
04能干什么？几个真实场景
📂
整理下载文件夹
你跟 Claude 说"帮我整理一下下载文件夹"，它会自动扫描所有文件，按类型分类，建好子目录，把文件移动进去。图片归图片、文档归文档、安装包归安装包。几秒钟的事。
📊
分析 Excel 数据
你丢一个 sales.csv 给 Claude，说"分析一下销售数据，告诉我前 10 大客户"。Claude 会用 Python 在内存里跑分析，然后直接给你看结果，还能生成图表。不需要打开 Excel，不需要写 Python 代码。
🔍
探索代码库
你接手一个项目，跟 Claude 说"帮我看看这个项目的结构"。它会递归扫描目录，理解项目架构，然后告诉你"这个项目是 Next.js 写的，主要逻辑在 src/app 下，API 路由在 src/api……"
🔗
远程 SSH 运维
你在 Claude 里说"SSH 连上服务器，检查一下磁盘空间，看看 Nginx 日志有没有报错"。Claude 会自动 SSH 连上去，执行命令，把结果返回给你。整个过程可交互，就像你在终端里操作一样。
05安全吗？
说到让 AI 控制你的电脑，安全肯定是第一关心的。Desktop Commander 有几种安全机制：
🛡️
命令黑名单
你可以配置哪些命令禁止执行（比如 blockedCommands），防止 Claude 跑危险操作。
📂
目录白名单
allowedDirectories 限制 Claude 只能操作你指定的文件夹。
🐳
Docker 完全隔离
Docker 安装方式下，Claude 在容器里运行，完全跟宿主机隔离。就算容器被搞坏了，也不影响你电脑上的任何东西。
📝
审计日志
所有操作都有日志记录，谁在什么时候调了什么工具，一清二楚。日志文件到了 10MB 会自动轮转。
⚠️ 注意
项目 README 里明确说了：allowedDirectories 的目录限制"目前只限制文件系统操作，不限制终端命令"。而且"目录限制和命令黑名单可以通过符号链接、命令替换、绝对路径等各种方式绕过"。所以如果你的电脑上有敏感数据，建议用 Docker 安装方式。别拿生产环境的机器瞎搞。
06Desktop Commander App（Beta）
除了 MCP 服务器，作者还做了一个独立的桌面应用（Beta 版），下载地址在 desktopcommander.app。这个 App 有啥特别的？
🧠
任意 AI 模型都能用
Claude、GPT-4.5、Gemini 2.5……你想用哪个用哪个，不绑定某一家。
👁️
文件变更实时预览
Claude 改文件的时候，你能看到实时变化，不是等它改完了再看结果。
🔌
自定义 MCP 和上下文
不需要写配置文件，直接在 App 里添加。
注意 MCP 服务器本身是独立工作的，你不用这个 App 也能用。App 只是多了一个更友好的界面。
