⭐白嫖党狂喜？这玩意儿是干啥的
好家伙，这项目有点猛。OmniRoute 是一个开源的 AI 网关，TypeScript 写的，MIT 协议。2026 年 2 月上线，到现在已经 12K+ Stars、1738 Forks，版本迭代到了 v3.8+，更新频率跟打了鸡血似的。
🤔
所以它到底是干嘛的？
说白了：你平时用 Claude Code、Cursor、Copilot 这些工具，每个都要配一个 API Key，用一家模型。OmniRoute 搭了一个中间层，给你一个统一的接口，背后自动路由到 231+ 家提供商——哪个便宜用哪个、哪个快用哪个、免费的优先用。甚至还能把 token 压缩到原来的 5%。
💡
跟 OpenRouter 啥区别？
OpenRouter 是个商业服务，你用它的 API 要付费。OmniRoute 是自己搭的网关，你部署在自己的机器上——完全免费，完全可控。而且它有 OpenRouter 没有的 token 压缩引擎和 MCP Server。
01一句话说清楚 OmniRoute
OmniRoute 是一个开源 AI 网关，暴露一个 OpenAI 兼容的 /v1/* 接口，背后聚合了 231+ 家模型提供商（50+ 家免费）。你用它连 Claude Code、Cursor、Cline、Copilot 等工具，它能自动选最优的模型、压缩 token、提供 MCP 和 A2A 能力。

02核心功能——让它值得 12K+ Stars
OmniRoute 不是一个"又一个 AI 网关"，它的几个核心功能确实有料：
🆓 50+ 免费模型提供商
不用花一分钱就能用的模型：Kiro（Claude Sonnet 每月 50 额度）、Qoder（kimi-k2-thinking 无限量）、Qwen（qwen3-coder 无限量）、Pollinations（GPT-5/Claude/Gemini 免 Key）、LongCat（每天 5000 万 token）、Cloudflare AI（50+ 模型）、NVIDIA NIM（129 模型）、Cerebras（每天 100 万 token）……全部免注册、免绑卡。
🗜️ 9 层压缩引擎——最高省 95% token
最炸裂的功能。OmniRoute 内置了 9 层压缩管线，层层叠加：会话去重 → CCR 归档 → RTK（工具输出智能过滤）→ Headroom 表格压缩 → Caveman（规则式散文压缩，省 65-75%）→ LLMLingua-2 语义剪枝 → Lite 空白修剪 → Aggressive 摘要+老化 → Ultra 启发式修剪。
一键预设：Lite（~15%）/ Standard（~30%）/ Aggressive（~50%）/ Ultra（~75%）/ RTK（60-90%）/ Stacked RTK+Caveman（78-95%）
🔄 14 种路由策略 + 4 级自动降级
14 种路由策略：优先级、加权、成本优化、上下文中继……默认的智能策略会根据你的需求自动选模型。如果当前模型挂了或不够用了，4 级降级：Subscription → API Key → Cheap → Free，保证请求不丢。
🔌 内置 MCP Server（37+ 工具）
OmniRoute 自带 MCP Server，支持 stdio/SSE/Streamable HTTP 三种传输协议，13 个作用域共 37+ 个工具。这意味着你可以把它当 MCP 中心用，其他 MCP 客户端直接连上来就能用。
03安装——选 npm 还是 Docker
两种方式，看哪个对你胃口：
方式 1：npm 全局安装（推荐本地开发用）
# 安装
npm install -g omniroute

# 启动
omniroute

# 然后打开 http://localhost:20128
启动后控制台会打印 Dashboard 地址和 API 端口。默认端口 20128。
方式 2：Docker（推荐 VPS 或长期运行）
docker pull diegosouzapw/omniroute:latest
docker run -d -p 20128:20128 --name omniroute diegosouzapw/omniroute
Docker 启动后一样，浏览器打开 http://localhost:20128。
⚠️ 踩坑提示
• npm 安装需要 Node.js 18+，先跑 node --version 确认
• 第一次启动 Dashboard 会引导你配置，但想跳过配置直接用？可以——什么都不配也能用，它会自动用内置的免费提供商（如 Pollinations）
• 如果要用自己的 API Key（比如 OpenAI/Anthropic），在 Dashboard → Providers 里添加即可
04连 Claude Code——实操演示
这是大家最关心的——怎么把 Claude Code 连到 OmniRoute 白嫖免费模型？
第一步：从 Dashboard 获取 Endpoint Key
打开 http://localhost:20128 → 左侧菜单 Endpoints → 复制你的 API Key（每个实例自动生成一个）。
第二步：配置 Claude Code
# 设置 Claude Code 使用 OmniRoute
claude config set proxyUrl http://localhost:20128
claude config set apiKey 你的EndpointKey
claude config set model auto
然后正常用 Claude Code，所有请求都会经过 OmniRoute。它会自动选一个可用的免费模型回你。
第三步：连 Cursor / Cline / Copilot
Cursor：Settings → Models → 添加自定义 Base URL → http://localhost:20128/v1，API Key 填 Endpoint Key
Cline：Provider 选 OpenAI Compatible → Base URL 同上
GitHub Copilot：Dashboard Endpoints 里有专门的 Copilot 配置方法

所有工具都把模型名设为 auto，OmniRoute 会自动路由到最合适的模型。
05压缩引擎怎么配
压缩是 OmniRoute 最亮的功能，配置也简单。在 Dashboard 里配一次，后面全自动：
Dashboard 配置步骤
1. 打开 Combos → + New Combo
2. 添加你的提供商链（比如 付费 → 免费）
3. 在 Compression 下拉选预设：
- Lite：日常不怕，~15% 省 token
- Standard (Caveman)：日常写代码，~30%
- Aggressive：长会话，~50%
- Ultra：极限省钱，~75%
- RTK：工具输出多的场景（shell/build/test），60-90%
- Stacked：RTK + Caveman 叠加，省 78-95%
4. 保存，之后所有请求自动压缩
📊 真实压缩效果
代码搜索（返回 100 条结果）：12345 token → 1543 token，省 87.5%
工具 JSON 输出：20123 token → 3218 token，省 84%
SRE 排障日志：60789 token → 8912 token，省 85.3%
20 轮对话历史：45678 token → 12345 token，省 73%
工具调用密集场景平均：~89%
06进阶——MCP 和 A2A
OmniRoute 不只是个网关，它还内置了 MCP Server 和 A2A 协议支持。
🔌
MCP Server（37+ 工具，3 种传输协议）
stdio / SSE / Streamable HTTP 三种传输，13 个 scope，涵盖文件系统、网络请求、搜索、代码分析等。你可以把 OmniRoute 当 MCP 中心用，其他 MCP 客户端直接连。
🤖
A2A（Agent-to-Agent 通信）
基于 JSON-RPC 2.0 + SSE，Agent 之间可以互相通信、协作。如果你的多个 AI Agent 需要对话，A2A 是基础设施。
07什么时候用 OmniRoute？
✅ 适合用它
• 想白嫖免费模型连到 Claude Code / Cursor 用
• 有多家模型 API Key，想要一个统一入口 + 智能路由
• token 消耗大户，想让压缩引擎帮你省钱的
• 需要自建 MCP Server 或 A2A 基础设施的
❌ 不太适合
• 只用一家模型、没有多提供商需求的用户
• 非常在意响应延迟的场景（加了网关层肯定比直连慢一点）
• 压缩引擎在某些场景下可能丢失信息（尤其 Ultra 模式）
