Docker 要慌了吗？
不一定，但这是 Apple 官方第一次出容器工具，意味着 macOS 上跑 Linux 容器有了"官方方案"。
💡
跟 Docker 最大区别：每容器一个独立 VM
Docker 在 Mac 上是在一个共享 Linux VM 里跑进程隔离。Apple Container 是每个容器一个独立轻量 VM——隔离性更强、也更"重"一点。
01一句话说清楚 Apple Container
Apple Container 是一个在 Mac 上跑 Linux 容器的 CLI 工具。它用 macOS 自带的 Virtualization.framework，每个容器启动一个独立的轻量虚拟机。Swift 编写，OCI 兼容——意味着你能用它 pull Docker Hub 上的镜像、build Dockerfile、push 到容器仓库。
架构对比：Docker vs Apple Container
🐳 Docker on Mac
你的 Mac
Hypervisor.framework
1 个共享 Linux VM
容器A | 容器B | 容器C（进程隔离）
🍎 Apple Container
你的 Mac
Virtualization.framework
容器A → 独立轻量VM
容器B → 独立轻量VM（硬件隔离）
技术栈分层
上层：container CLI
你敲的命令：container run、container build、container ls……
中层：container-apiserver 守护进程
后台服务，管理镜像、容器生命周期、网络
底层：Containerization Swift Package
Swift 包，封装 Virtualization.framework 提供 VM/镜像/进程管理 API
02快速上手——跟 Docker 几乎一样的用法
如果你用过 Docker，那上手 Apple Container 毫无门槛——命令风格几乎一模一样：
DockerApple Container
docker pscontainer ls
docker run nginxcontainer run nginx:latest
docker build .container build --tag my-app .
docker exec -it bashcontainer exec -it my-nginx bash
docker imagescontainer images list
docker logincontainer registry login
📌 安装方式
# Homebrew（最简单）
brew install container
也可以从 GitHub Releases 下载签名安装包，或从源码编译
跑一个 Nginx 容器只要 4 步：
# 1. 启动系统服务
container system start

# 2. 拉取镜像
container images pull nginx:latest

# 3. 运行容器
container run --rm --name my-nginx nginx:latest

# 4. 查看运行中的容器
container ls

05什么时候用 Apple Container？
✅ 适合用它
• 你是 M 芯片 Mac + macOS 26，想用 Apple 原生方案
• 需要硬件级隔离的场景（安全敏感、多租户）
• 想做 Apple 生态的原生容器开发（Swift Package）
• 喜欢尝鲜，想体验 Apple 版 OCI 运行时
❌ 暂时别用它
• 项目还没升级 macOS 26（macOS 15 网络功能受限）
• 需要生产级稳定性的团队（Docker 生态更强）
• Intel Mac 用户（不支持）
• 需要 Docker Compose / Swarm / K8s 集成（目前没有）
