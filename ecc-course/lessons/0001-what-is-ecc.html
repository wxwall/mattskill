01ECC 是什么？为什么是"操作系统"？
ECC 的全称是 Everything Claude Code。虽然名字带"Claude Code"，但它早已超越单一平台，成为一套跨 7 个 AI 编程助手的通用 Agent 配置与自动化系统。
把它理解为"操作系统"是最精准的比喻：
1
裸机 AI 助手 = 没有 OS 的电脑
每次对话从零开始，没有代码规范记忆，没有自动化，没有安全策略。同一类错误反复踩，每次重新教它规则。MCP 服务器乱开，上下文被无意中吃掉。
2
ECC = 装上了 OS 的电脑
自动记忆项目规则；一键触发代码审查/测试/构建修复；67 个专业子 Agent 随时待命；跨对话持久化学习；Token 优化内置。
3
10 个月实战打出来的，不是设计出来的
作者 @affaan 从 Claude Code 实验版就开始深度使用，在多个真实产品中反复验证。还拿了 Anthropic × Forum Ventures 黑客松冠军（2025.09），用 Claude Code 从零建了 zenith.chat。

02五大核心模块：拆解 ECC 的内部结构
ECC 不是一个大杂烩，它有清晰的五层模块化设计：
🧠 Agents — 67 个专业子 Agent，解决"AI 什么都懂但什么都不精"
把复杂任务委派给专门 Agent。代码审查 → code-reviewer，安全检查 → security-reviewer，构建错误 → build-error-resolver，架构设计 → architect，文档更新 → doc-updater，端到端测试 → e2e-runner。每种语言还有专属 Agent：python-reviewer、go-reviewer、typescript-reviewer、java-reviewer、kotlin-reviewer、rust-reviewer、cpp-reviewer、fsharp-reviewer、database-reviewer、mle-reviewer。甚至还有一个 chief-of-staff 负责沟通分诊。
📋 Skills — 277 个可复用技能模块，你的 AI "应用商店"
ECC 最主要的交互界面。每个 Skill 是一个 markdown 文件，定义了特定任务的完整工作流。分为几大类：
• 语言模式：Python、Go、TypeScript、Java、Swift、Perl、Rust、C++、PHP、Kotlin 等 12+ 语言的编码规范和测试流程
• 框架模式：Django、Spring Boot、Laravel、Quarkus、NestJS、Next.js、PyTorch
• 工作流：TDD 开发、验证循环、安全审查、端到端测试、搜索优先
• 业务技能：文章写作、内容引擎、市场调研、投资人材料、品牌语音
• 运营技能：并行执行优化器、基准优化循环、延迟关键系统
• 领域技能：ClickHouse 分析、视频处理、MCP 服务器模式、数据库迁移、部署模式、API 设计、Docker
⚡ Hooks — 事件驱动自动化，给 AI 装上"条件反射"
在工具调用前后自动触发：PreToolUse 拦截危险命令（如 rm -rf、force push），SessionStart 自动加载上次上下文，SessionEnd 保存学习成果，Stop 触发压缩前检查。支持环境变量精细化控制：ECC_HOOK_PROFILE=strict 严格模式、ECC_DISABLED_HOOKS 禁用指定 Hook、ECC_SESSION_START_MAX_CHARS 上下文长度上限。
📐 Rules — 强制性规范，AI 的"法律"
始终生效的规则，分为：通用规则（编码风格、Git 工作流、测试覆盖率 ≥80%、性能、设计模式、Hooks 规范、Agent 规范、安全）和 语言专属规则（TypeScript、Python、Go、Swift、PHP、ArkTS/HarmonyOS）。Rules 和 Skills 的区别：Rules 是"法律"（永远生效），Skills 是"工具书"（需要时调用）。
💾 Memory Persistence — 跨对话记忆，AI 永不"失忆"
Hooks 自动保存和加载上下文：会话摘要、学习到的技能、会话别名、指标数据全部持久化到本地。支持按平台隔离（Cursor 一套、Claude Code 一套）。Continuous Learning v2 是明星功能——自动从对话中提取模式，带置信度评分，通过 /evolve 命令聚合成新的 Skill。数据存放在 ECC_AGENT_DATA_HOME（默认 ~/.claude），30 天保留窗口。

04安装 ECC：三种方式，选一种就好
ECC 提供三种安装路径。最重要的一条规则：不要叠加安装方式！最常见的翻车就是先插件装完又跑 shell 脚本。
⭐ 推荐：插件安装（Claude Code 用户首选）
在 Claude Code 终端直接输入：
/plugin install ecc@ecc
从官方市场自动拉取，开箱即用。安装后自动拥有所有 commands 和 skills。
📦 手动安装：Shell 脚本（精细控制）
install.sh --profile full 全量安装所有组件。
install.sh --profile minimal 只装核心。还有 v1.9.0 的选择性安装管道（manifest-driven），可按需挑选。
📦 npm 安装（跨平台通用）
npx ecc-install 一键安装。
npx ecc consult "如何设计 REST API" 直接咨询 ECC 知识库。
⚠️ 插件用户的额外步骤
Claude Code 插件不能自动分发 Rules。插件用户需要手动复制需要的规则目录到 ~/.claude/rules/。这是 Claude Code 插件系统的限制，不是 ECC 的 bug。

07五大杀手级特性：为什么 ECC 不只是"另一个工具集"
🎯
Token 优化开箱即用
ECC 推荐：默认模型用 Sonnet、MAX_THINKING_TOKENS=10000（默认 31999）、CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=50（默认 95）。最关键的警告：开启太多 MCP 服务器会让 200K 上下文窗口缩水到 ~70K。建议保持 10 个 MCP 以下、80 个工具以内。
🔐
AgentShield 安全引擎
ECC 的独立安全工具：1282 个测试、98% 覆盖率、102 条静态分析规则。扫描 Claude Code 配置中的漏洞、错误配置、注入风险。--opus 标志启动红队/蓝队/审计三方对抗（3 个 Claude Opus 4.6 Agent 同时跑）。输出 A-F 等级。CI 中 exit code 2 可直接作为构建门禁。
🔄
工作树并行化 + 多 Agent 编排
Git worktree + cascade 方法让多个 Agent 并行工作不冲突。v2.0.0 的 orch-* 系列编排器支持工作树生命周期管理。PM2 多 Agent 工作流，适合大型项目的并行开发。
🧠
Continuous Learning v2：AI 越用越聪明
不只是"记住上次对话"——v2 系统带置信度评分，自动从对话中提取模式成为"本能"（instinct）。通过 /evolve 命令将本能聚类成新 Skill。本能可以导入/导出/团队共享。这是 ECC 最与众不同的一点：你的配置不是死的，而是活的。
💰
MIT 开源 + 免费永远 + 单人维护周更 7 平台
ECC 开源部分永久 MIT 协议，自由使用和修改。ECC Pro 是 GitHub App，$19/人/月，面向私有仓库的高级功能（Skill Creator 高级版等）。赞助者和 Pro 订阅者的钱支撑着一个独立维护者每周在 7 个平台上同步发布——这不是 VC 烧钱的项目，是一个开发者在火线上打了 10+ 个月仗沉淀出来的。

09上手实操：从安装到第一个安全扫描
以下是在 Claude Code 上的完整上手流程：
第 1 步：安装
/plugin install ecc@ecc
等待安装完成，自动加载所有 commands 和 skills。
第 2 步：复制规则（插件用户必须手动做这一步）
cp -r ~/.claude/plugins/ecc@ecc/rules/common ~/.claude/rules/
cp -r ~/.claude/plugins/ecc@ecc/rules/typescript ~/.claude/rules/
选择你需要的语言规则目录复制。不需要全部。
第 3 步：运行安全扫描
/security-scan
AgentShield 会扫描你的 Claude Code 配置，输出 A-F 安全评级。首次运行建议加 --opus 深度扫描。
第 4 步：体验代码审查
/code-review
对当前变更进行多维度审查，子 Agent 自动介入。
第 5 步：启用持续学习
/learn
此后 ECC 会自动从你的每次对话中提取模式，带置信度评分存储。
