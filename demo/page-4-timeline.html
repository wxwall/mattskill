STEP 1

你需要准备什么

CodeGraph 自带运行时，不需要安装 Node.js。它把自己的 Node.js 打包进去了——下载解压就能用，不需要编译、不需要原生构建、所有平台行为一致。

操作系统
macOS · Linux · Windows
x64 & arm64
支持的 Agent
Claude · Cursor · Codex
Gemini · Antigravity · Kiro
项目要求
任何语言项目
24 语言 · 17 框架路由
STEP 2

安装 CLI

三种方式，任选一种：

✅ 推荐
一键安装脚本
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
macOS / Linux。Windows 请用 PowerShell 版本
npm 全局安装 （需要 Node.js）
npm i -g @colbymchenry/codegraph
✓ 验证安装： codegraph --version → v1.1.0 或更高
💡 之后随时升级：codegraph upgrade — 自动检测安装方式并原地更新
STEP 3

接入 AI Agent

安装完 CLI 后，需要把 CodeGraph 的 MCP Server 配置写入 Agent 的配置文件里。这一步不会索引任何代码——只是告诉 Agent「有 CodeGraph 这个工具可以用」。

codegraph install
1
🔍 自动检测
扫描你机器上安装了哪些 Agent（Claude Code、Cursor、Codex 等），自动写入对应的 MCP 配置。
2
📝 写入配置
在 Agent 的 MCP config 中添加 CodeGraph Server 条目，包含 stdio 启动命令和工具权限。
3
📋 注入指令
在 CLAUDE.md / AGENTS.md 中插入一段 marker-fenced 指令，告诉子 Agent 也可以用 codegraph explore CLI。
📄 MCP 配置示例（以 Claude Code 为例）
{
  "mcpServers": {
    "codegraph": {
      "type": "stdio",
      "command": "codegraph",
      "args": ["serve", "--mcp"]
    }
  }
}

🔗
解析符号引用关系
跨文件 + 框架路由
↓
💾
写入 codegraph.db
SQLite + FTS5
↓
✅
完成！文件监听器已启动
后续自动同步
STEP 5

文件结构与日常使用

📁 生成的文件
your-project/
├── .codegraph/
│ ├── codegraph.db    ← SQLite 数据库（图数据）
│ └── codegraph.json  ← 项目级配置（可选）
└── ...
💡 建议：把 .codegraph/ 加入 .gitignore
索引文件不提交到 Git——它是派生数据，团队成员各自在自己机器上运行 codegraph init 即可。
🔄
自动同步
init 之后不需要手动操作。文件监听器检测到变化 → debounce 2 秒 → 增量更新索引。
🧹
卸载
codegraph uninstall 从 Agent 配置中移除 · codegraph uninit 删除项目索引
🏗️
Monorepo
在根目录运行 codegraph init，所有子项目一起索引。
STEP 6

验证是否生效

在 Claude Code 里问一个结构性问题，看 Agent 是否使用了 CodeGraph 而不是 grep：

❌ 没装 CodeGraph
grep("createUser")
Read(controller.ts)
Read(service.ts)
Read(repository.ts)
… 多次工具调用
✅ 装了 CodeGraph
codegraph_explore(
  "createUser 调用链"
)
→ 1 次调用，完整结果
🔧 手动检查索引状态： codegraph status → 显示索引了多少文件、符号、关系边
