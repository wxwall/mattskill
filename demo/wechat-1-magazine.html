COURSE 0003
安装与初始化
CodeGraph
CodeGraph 自带运行时
不需要安装 Node.js
⏱ 15 分钟 ← 第 2 课

CodeGraph 自带运行时，不需要安装 Node.js。它把自己的 Node.js 打包进去了——下载解压就能用，不需要编译、不需要原生构建、所有平台行为一致。

1 你需要准备什么
🖥️
操作系统
macOS · Linux · Windows / x64 & arm64
🤖
支持的 Agent
Claude Code · Cursor · Codex CLI · Gemini · Kiro
📦
项目条件
任何语言 · 24 种语言 · 17 种框架路由
2 安装 CLI

三种方式，任选一种：

✅ 推荐 · 一键安装脚本
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
macOS / Linux。Windows 用 PowerShell 版本
npm 全局安装 （需 Node.js）
npm i -g @colbymchenry/codegraph
✅ 验证安装： codegraph --version → v1.1.0+
💡 随时升级：codegraph upgrade 自动检测安装方式并原地更新
3 接入 AI Agent

安装完 CLI 后，需要把 CodeGraph 的 MCP Server 配置写入 Agent 的配置文件。这一步不会索引任何代码——只是告诉 Agent「有 CodeGraph 这个工具可以用」。

codegraph install
1
🔍 自动检测 — 扫描机器上安装了哪些 Agent
2
📝 写入配置 — MCP config 添加 CodeGraph Server
3
📋 注入指令 — CLAUDE.md 插入 marker-fenced 指令
📄 以 Claude Code 为例，安装后实际写入的 MCP 配置
{
  "mcpServers": {
    "codegraph": {
      "type": "stdio",
      "command": "codegraph",
      "args": ["serve", "--mcp"]
    }
  }
}
同时自动设置 mcp__codegraph__* 通配符权限，调用时不需要每次弹确认框

5 生成的文件与日常使用
your-project/
├── .codegraph/
│ ├── codegraph.db   ← SQLite 数据库
│ └── codegraph.json  ← 项目配置（可选）
└── ...
⚠️ 建议：把 .codegraph/ 加入 .gitignore
索引文件不提交到 Git——它是派生数据，团队成员各自在自己机器上运行 codegraph init 即可。
🔄
自动同步
文件变化 → debounce 2s → 增量更新。写完代码保存，图就更新了。
🧹
卸载
codegraph uninstall 移除 Agent 配置 · codegraph uninit 删除索引
🏗️
Monorepo
根目录运行 init，所有子项目一起索引。Agent 通过 projectPath 指定目标。
6 验证是否生效

在 Claude Code 里问一个结构性问题，看 Agent 是否使用了 CodeGraph 而不是 grep：

grep("createUser")
Read(controller.ts)
Read(service.ts)
Read(repository.ts)
… 多次工具调用
↧
✅ 装了 CodeGraph
codegraph_explore(

)
→ 1 次调用，完整结果
🔧 手动检查： codegraph status → 显示索引了多少文件、符号、关系边

小测验
问题 1：codegraph install 和 codegraph init 的区别是什么？
A. 没有区别，都是安装 CodeGraph
✓ B. install 把 MCP Server 配置写入 Agent，init 构建项目代码图谱
C. install 安装 CLI 工具，init 初始化 Git 仓库
D. install 下载 CodeGraph，init 启动后台服务
问题 2：为什么建议把 .codegraph/ 加入 .gitignore？
A. 因为包含密码等敏感信息
✓ B. 索引是派生数据，团队成员各自运行 init 生成即可
C. 因为文件太大，Git 无法处理
D. 官方不允许提交索引文件
