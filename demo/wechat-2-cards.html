安装与初始化
CodeGraph
自带运行时 · 无需 Node.js · 解压即用
← 第 2 课

CodeGraph 把自己的 Node.js 打包进去了——下载解压就能用。不需要编译、不需要原生构建、所有平台行为一致。

STEP 1
你需要准备什么
🖥️
操作系统
macOS (x64, arm64) · Linux · Windows
🤖
支持的 Agent
Claude Code · Cursor · Codex · Gemini · Kiro
📦
项目条件
任何语言 · 24 种语言 · 17 种框架路由
STEP 2
安装 CLI
三种方式，任选一种
✅ 推荐 · 一键安装
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
npm 全局安装 需 Node.js
npm i -g @colbymchenry/codegraph
✅ codegraph --version → v1.1.0+
💡 codegraph upgrade 自动检测并原地更新
STEP 3
接入 AI Agent

把 CodeGraph 的 MCP Server 配置写入 Agent 的配置文件。这一步不会索引任何代码。

codegraph install
1
🔍 自动检测 — 扫描机器上安装的 Agent
2
📝 写入配置 — MCP config 添加 Server
3
📋 注入指令 — CLAUDE.md 插入指令
📄 实际写入的 MCP 配置
{
  "mcpServers": {
    "codegraph": {
      "type": "stdio",
      "command": "codegraph",
      "args": ["serve", "--mcp"]
    }
  }
}
STEP 4
初始化项目
cd your-project && codegraph init
🔍 扫描项目文件 跳过 node_modules
↓
🌳 tree-sitter 解析 → 提取符号 24 种语言
↓
🔗 解析符号引用关系 跨文件 + 框架路由
↓
💾 写入 .codegraph/codegraph.db SQLite + FTS5
↓
✅ 完成！文件监听器已启动 后续自动同步

STEP 6

❌ 没装 CodeGraph
grep("createUser")

Read(service.ts)
Read(repository.ts)

↧
✅ 装了 CodeGraph
codegraph_explore(

小测验
Q1：codegraph install 和 codegraph init 的区别？
A. 没有区别，都是安装 CodeGraph
✓ B. install 配置 Agent，init 构建代码图谱
C. install 安装 CLI 工具，init 初始化 Git 仓库
D. install 下载 CodeGraph，init 启动后台服务
Q2：为什么把 .codegraph/ 加入 .gitignore？
A. 包含密码等敏感信息
✓ B. 派生数据，成员各自 init 生成
C. 文件太大，Git 无法处理
D. 官方不允许提交索引文件
