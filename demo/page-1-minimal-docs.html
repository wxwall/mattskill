安装与初始化 CodeGraph

CodeGraph 自带运行时，不需要安装 Node.js。下载解压就能用——不需要编译、不需要原生构建、所有平台行为一致。

1. 你需要准备什么

🖥️
操作系统
macOS (x64, arm64)
Linux (x64, arm64)
Windows (x64, arm64)
🤖
支持的 Agent
Claude Code · Cursor
Codex CLI · Gemini CLI
Antigravity · Kiro
📦
项目条件
任何语言项目
24 种语言
17 种框架路由

2. 安装 CLI

三种方式，任选一种：

✅ 推荐
一键安装脚本
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
macOS / Linux。Windows 用 PowerShell 版本
npm 全局安装 （需要 Node.js）
npm i -g @colbymchenry/codegraph

验证安装：codegraph --version → v1.1.0 或更高

3. 接入 AI Agent

安装完 CLI 后，把 CodeGraph 的 MCP Server 配置写入 Agent 的配置文件：

codegraph install
1 自动检测 — 扫描机器上安装了哪些 Agent
2 写入配置 — 在 Agent 的 MCP config 中添加 CodeGraph Server
3 注入指令 — 在 CLAUDE.md 中插入 marker-fenced 指令

4. 初始化项目

cd your-project
codegraph init
🔍 扫描项目文件 — 自动跳过 node_modules 等
🌳 tree-sitter 解析 → 提取符号 — 24 种语言
🔗 解析符号引用关系 — 跨文件 + 框架路由
💾 写入 .codegraph/codegraph.db — SQLite + FTS5
✅ 完成！文件监听器已启动 — 后续自动同步

5. 生成的文件

codegraph init 只在你的项目里创建一个目录：

your-project/
├── .codegraph/
│ ├── codegraph.db       ← SQLite 数据库（图数据）
│ └── codegraph.json     ← 项目级配置（可选）
└── ...
💡 建议：把 .codegraph/ 加入 .gitignore，索引文件不提交到 Git

6. 自动同步与日常使用

🔄
自动同步
init 之后不需要手动操作。文件监听器检测到变化 → debounce 2 秒 → 增量更新索引
🧹
卸载
codegraph uninstall 从 Agent 配置中移除，codegraph uninit 删除项目索引
🏗️
Monorepo
在根目录运行 codegraph init，所有子项目一起索引

7. 验证是否生效

在 Claude Code 里问一个结构性问题，看 Agent 是否使用了 CodeGraph 而不是 grep：

❌ 没装 CodeGraph
grep("createUser")
Read(controller.ts)
Read(service.ts)
Read(repository.ts)
… 多次工具调用
✅ 装了 CodeGraph
codegraph_explore(
  "createUser 调用链"
)
→ 1 次调用，完整结果
