CodeGraph 架构概览

📋 关联课程：第 1 课 📦 版本：v1.1.0 🔄 更新：2026-06-23

四阶段流水线

阶段工具输入输出
1. Extract tree-sitter 源码文件（24 种语言） AST → 符号（函数、类、方法）+ 关系（调用、导入、继承）
2. Store SQLite + FTS5 提取的符号和关系 .codegraph/codegraph.db
3. Resolve CodeGraph 解析引擎 符号引用 + 导入关系 调用图、继承链、框架路由映射、跨语言桥接
4. Serve MCP Server Agent 结构化查询 源码片段 + 调用链 + 影响分析

核心概念

Node（节点）

代码中的可识别实体：函数、方法、类、接口、常量、路由处理器、Vuex mutation、Pinia store 等。每个节点代表一个可以"被调用"、"被引用"、"被继承"的符号。

Edge（边）

节点之间的关系：

  • calls — 函数 A 调用函数 B
  • imports — 文件 A 导入模块 B
  • extends / implements — 继承与接口实现
  • routes — URL 模式 → 处理函数（框架感知）
  • bridges — 跨语言调用（Swift↔ObjC、React Native 桥）

Knowledge Graph（知识图谱）

节点 + 边的整体结构。不是扁平的符号表，而是表达代码拓扑结构的有向图。Agent 可以在图上做"从这个函数出发，追踪所有下游调用"等图遍历操作。

MCP 工具集

工具用途默认
codegraph_explore主工具：一次查询 → 入口点 + 相关符号 + 调用流 + 影响范围✅
codegraph_node获取单个符号详情，可读完整源文件❌ 隐藏
codegraph_search全文搜索整个代码库❌ 隐藏
codegraph_callers谁调用了这个符号❌ 隐藏
codegraph_callees这个符号调用了谁❌ 隐藏
codegraph_impact变更影响范围分析（修改此符号会影响什么）❌ 隐藏
codegraph_files列出项目中已知文件❌ 隐藏
codegraph_status索引状态检查❌ 隐藏
设计哲学

一个强工具（codegraph_explore）比一堆窄工具更能引导 Agent 正确使用。其他工具可通过 CODEGRAPH_MCP_TOOLS 环境变量重新启用。

三层新鲜度保证

  1. 文件监听器 — OS 原生事件（FSEvents / inotify）→ debounce 后增量同步
  2. 过期警告 — debounce 窗口期内，对修改中的文件返回 staleness banner
  3. 重连校验 — MCP 重连时，用 (size, mtime) + 内容哈希做快速增量对比
