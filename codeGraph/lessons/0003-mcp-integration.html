01什么是 MCP
MCP（Model Context Protocol）是 Anthropic 推出的开放协议，定义了 AI Agent 与外部工具之间的通信标准。你可以把它理解为"AI 界的 USB 协议"——统一了工具接入方式。
🔌
MCP Server
工具提供方。CodeGraph 启动一个 MCP Server，把自己的代码知识图谱暴露给 AI Agent。
🤖
MCP Client（AI Agent）
工具消费方。Claude Code、Cursor 等 AI 编程助手作为 Client，连接到 CodeGraph Server 查询代码信息。
📌 类比理解：没有 MCP 时，每个 AI Agent 需要自己写自定义集成。有了 MCP，CodeGraph 只要实现一次 Server 端，所有支持 MCP 的 AI 工具都能直接使用。
02在 Claude Code 中配置 CodeGraph
在 Claude Code 的配置文件 claude.json 中添加 MCP Server 配置：
{
  "mcpServers": {
    "codegraph": {
      "command": "codegraph",
      "args": ["serve", "--mcp"],
      "cwd": "/path/to/your/project"
    }
  }
}
📍
command 和 args 告诉 Claude Code 如何启动 CodeGraph 的 MCP 服务
📍
cwd 指定项目路径，CodeGraph 会读取该项目的 .codegraph/ 索引
配置好后重启 Claude Code，CodeGraph 的 MCP 工具会自动注册。你可以通过 /mcp 命令查看已连接的 MCP Server 列表。
03codegraph_explore —— 主工具
CodeGraph 暴露了多个 MCP 工具，其中 codegraph_explore 是主工具。它设计为一次调用就能提供 Agent 需要的全部上下文：
🎯
入口点识别
根据用户的任务描述，找到最相关的函数/类/文件作为起点
🔗
相关符号列表
展示与入口点相关的所有符号及其文件位置
🌊
调用流
展示函数之间的调用链——谁调了谁，调用路径是什么
📊
影响范围
如果修改这个入口点，哪些代码会被影响
📌 设计哲学：一个强工具（codegraph_explore）比一堆窄工具更能引导 Agent 正确使用。其他工具（node / search / callers / callees / impact）默认隐藏，可通过 CODEGRAPH_MCP_TOOLS 环境变量重新启用。
04实际使用效果对比
以一个实际场景为例：你问 Agent"帮我修改用户登录的密码验证逻辑"。
❌ 没有 CodeGraph
1. Agent 搜索文件名 → 找到 auth.ts
2. Agent 读取整个 auth.ts → 找到 login 函数
3. Agent 读取 User 模型 → 理解用户表结构
4. Agent 读取 password utility → 了解加密方式
5. Agent 开始修改
总计：4-6 次文件读取，大量 token 消耗
✅ 使用 CodeGraph
1. Agent 调用 codegraph_explore("password verification")
2. MCP 返回：login 函数位置、User 模型、password utility、调用链
3. Agent 直接读取关键源码片段 → 开始修改
总计：1 次 MCP 查询 + 0-1 次针对性读取，token 消耗减少 47%
