01框架感知路由
CodeGraph 内置了 17 种框架的路由识别器，能自动从框架的 URL 模式映射到处理函数。AI Agent 可以直接问"找到 /api/users 的路由处理器"而无需手动搜索。
ExpressNestJSFastAPIDjangoFlaskSpringGinAxumASP.NETLaravelRailsVaporReact RouterSvelteKitVue/NuxtGoFrame
例如，对于 FastAPI 中的路由：
# FastAPI 路由定义
@router.get("/api/users/{user_id}")
async def get_user(user_id: int):
    return await db.fetch_user(user_id)
CodeGraph 会自动建立 GET /api/users/{user_id} → get_user 的映射关系。Agent 可以查询"这个 API 的路由在哪"并直接获得答案。
02三层新鲜度保证
代码在变化，索引必须跟上。CodeGraph 提供了三层机制确保索引始终最新：
👀
文件监听器
使用 OS 原生事件（FSEvents / inotify / ReadDirectoryChangesW）监听文件变化，变化发生后 debounce 后自动增量同步。运行 codegraph watch 启动。
⚠️
过期警告
在 debounce 窗口期内，对正在修改中的文件返回 staleness banner，告知 Agent 该文件可能不是最新状态。
🔄
重连校验
MCP 重连时，用 (size, mtime) + 内容哈希做快速增量对比，只重索引有变化的文件。
03安全与隐私配置
🛡️
100% 本地运行
所有数据都存储在本地 SQLite 中，从不离开你的机器。无需任何 API Key，没有外部请求。
🔐
敏感文件保护
Spring 配置文件中的密码值不索引（只索引 key）。.gitignore 中的文件自动跳过。
🔗
路径穿越防护
符号链接不会暴露项目目录外的文件，防止意外索引敏感系统文件。
📄
自定义忽略规则
在 .codegraphignore 文件中定义额外忽略规则，格式同 .gitignore。
04最佳实践清单
✅
首次索引后启动 watch 模式
开发过程中运行 codegraph watch，确保索引始终与代码同步。
✅
CI/CD 中自动化索引
在 CI 流水线中加上 codegraph index 步骤，确保代码评审时 Agent 有最新索引可用。
✅
将 .codegraph/ 加入 .gitignore
数据库文件不需要入版本控制，每个开发者本地自行索引即可。CodeGraph init 后会自动添加。
✅
多项目共享配置
团队可在项目根目录放一份统一的 .codegraph/codegraph.toml，确保所有成员配置一致。
✅
大项目按模块分类索引
对于 monorepo 或多模块项目，可为每个子项目单独运行 codegraph init 并分别配置 MCP。
