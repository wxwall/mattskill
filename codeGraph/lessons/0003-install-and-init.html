安装与初始化 CodeGraph

📚 课程 0003 ⏱ 预计 10 分钟 ← 第 2 课

1. 安装 CLI

不需要 Node.js——CodeGraph 自带运行时。macOS / Linux 一条命令：

curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh

Windows (PowerShell)：irm .../install.ps1 | iex，或用 npm：npm i -g @colbymchenry/codegraph。

装完开新终端验证：

codegraph --version  # → v1.1.0

升级用 codegraph upgrade，自动检测安装方式原地更新。

2. 接入 Agent

这一步不索引代码——只把 CodeGraph 的 MCP Server 配置写入 Agent：

codegraph install

自动检测已安装的 Agent（Claude Code、Cursor、Codex、Gemini CLI 等），写入 MCP 配置 + CLAUDE.md 指令片段，同时设置 mcp__codegraph__* 权限避免每次弹确认框。

实际写入的 MCP 配置：

{
  "mcpServers": {
    "codegraph": {
      "type": "stdio",
      "command": "codegraph",
      "args": ["serve", "--mcp"]
    }
  }
}

3. 初始化项目

cd your-project
codegraph init

init 执行完整流水线：

🔍 扫描项目文件 跳过 node_modules 等
↓
🌳 tree-sitter 提取符号 24 种语言
↓
🔗 解析引用 + 框架路由 跨文件关联
↓
✅ 写入 .codegraph/codegraph.db 文件监听已启动

init 之后不需要手动操作。文件监听器检测变化 → debounce 2 秒 → 增量更新。

4. 日常管理

.gitignore
.codegraph/

索引是派生数据，不提交。团队成员各自 init。

查看状态
codegraph status

索引了多少文件、符号、关系边。

卸载
codegraph uninstall

从 Agent 配置中移除。uninit 删索引。

Monorepo

根目录 init，子项目通过 projectPath 参数区分。
