# /goal 速查卡 ## 一句话 `/goal <条件>` → Claude 自动循环干活，条件满足自动停。 ## 命令速查 | 命令 | 作用 | |------|------| | `/goal <条件>` | 设目标，开始自主执行 | | `/goal` | 查看当前状态（条件、轮次、token） | | `/goal clear` | 取消当前目标 | ## 条件万能模板 ``` /goal [干什么活] 直到 [可测量的终点] 且不 [约束条件] ``` ## 好条件三要素 1. **可测量的终点** — 不是"感觉好了"，而是可验证的结果 2. **能在对话中证明** — 评估器只看对话记录 3. **有边界约束** — 防止 Claude 跑偏 ## 适合的场景 - 修测试（跑到全通过） - 批量迁移（class → function 组件） - 生成文件（CHANGELOG、README） - CI 流水线（`claude -p "/goal ..."`） ## 不适合的场景 - 主观判断任务（"让 UI 好看点"） - 一次性 30 秒小编辑 - 架构决策（每个分叉需要你把关） - 第一次碰的项目（Claude 缺领域上下文） - 破坏性操作难以撤销时 ## 黄金组合 `/goal` + Auto mode = 完全自动驾驶（去掉工具确认 + 去掉轮次确认） ## 最佳套路 1. 先 `/plan` 设计方案，验收标准变 /goal 条件 2. 条件里要求维护 `progress.md`（审计记录） 3. 加轮次上限兜底（"或最多 20 轮后停止"）
