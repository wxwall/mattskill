# Mission: Goal Mode

## Why
我想搞懂 Claude Code 的 `/goal` 命令到底是个啥、怎么用、什么时候该用。目前我是 Claude Code 的日常用户，已经习惯了手动驱动每一轮对话——发指令、看结果、再发指令。但如果 Claude 能自己连续干活，我不需要每轮都盯着，那效率能提一大截。

## Success looks like
- 能用自己的话讲清楚 `/goal` 的运作原理（循环、评估器、停止条件）
- 会写有效的 goal 条件——知道好条件长啥样、烂条件长啥样
- 知道什么时候用 `/goal`、什么时候别用
- 能在日常项目中实际用起来，感受到真实的效率提升
- 了解 `/goal` vs `/loop` vs Auto mode 的区别和组合玩法

## Constraints
- 学习时间：业余时间，每次学一个短课程即可
- 语言偏好：中文教学，技术术语保留英文
- 当前环境：已安装 Claude Code v2.1.139+

## Out of scope
- 不涉及 Claude Code 插件开发
- 不深入研究 evaluator model 的内部实现
- 不涉及 `/loop`、stop hooks 等其他自主模式的深度对比（可后续扩展）
