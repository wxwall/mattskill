# 学习记录：/goal 基础概念

## 日期
2026-07-06

## 学到了什么
`/goal` 是 Claude Code 的"自动挡"模式——设定一个可验证的完成条件后，Claude 自动循环干活直到条件满足。

核心机制是两个东西打配合：
1. **自动循环**：每轮结束后自动开下一轮（不需要你 re-prompt）
2. **轻量评估器**（默认 Haiku）：每轮结束扫一眼对话记录，判断条件是否满足

关键洞察：评估器只看对话记录中已有的内容，不会自己跑命令。所以条件必须是 Claude 能在对话中证明的——"测试退出码为 0"可以，"代码好看多了"不行。

## 为什么重要
这是 Claude Code 从"你驱动的工具"变成"自主干活的 Agent"的关键能力。跟 Auto mode 搭配后，就是真正的自动驾驶：Auto mode 去掉工具调用确认，/goal 去掉每轮对话确认。

## 相关知识
- `/goal` vs `/loop`：/goal 是"干到条件满足自动停"，/loop 是"按时间间隔定期触发"
- `/goal` vs stop hook：stop hook 更灵活（自定义停止逻辑），/goal 更简单

## 何时复习
下次准备在实际项目里用 /goal 时重读——尤其注意好条件的写法和不适合的场景。
