01一键安装
no-mistakes 是用 Go 编写的静态二进制，安装方式极其简单。最推荐的方式是 curl 一键安装：
🌐 curl 安装（macOS/Linux）
curl -fsSL https://raw.githubusercontent.com/kunchenguid/no-mistakes/main/docs/install.sh | sh
🪟 Windows（PowerShell）
powershell -c "irm https://raw.githubusercontent.com/kunchenguid/no-mistakes/main/docs/install.ps1 | iex"
🔧 源码构建（需要 Go）
git clone https://github.com/kunchenguid/no-mistakes.git
cd no-mistakes && make build
📌 安装验证：运行 no-mistakes version 查看安装版本号。v1.31.2 是 2026 年 6 月的最新版本。
02初始化项目
安装完成后，进入项目目录运行初始化命令：
cd your-project
no-mistakes init
初始化会做三件事：
1
创建门禁仓库
在 ~/.no-mistakes/repos/ 下创建一个 bare 仓库作为本地门禁。
2
检测远程仓库
自动检测当前项目的 origin remote，将远程指向它。
3
安装 Agent Skill
安装 /no-mistakes agent skill，支持在 Claude Code 等工具中通过命令触发门禁。
📌 Fork 贡献：如果为上游仓库做贡献，用 no-mistakes init --fork-url <你的-fork-url>。origin 保持指向上游仓库。
03三种触发方式
初始化完成后，有三种方式可以触发 no-mistakes 的验证流水线：
方式一：git push no-mistakes
最直接的触发方式。已提交的代码直接推送到门禁 remote：
git push no-mistakes <branch>

适用于：已经本地提交完成，只需要过一遍门禁的场景。推送到门禁后，可以运行 no-mistakes 打开 TUI 查看流水线状态。
方式二：no-mistakes（TUI）
直接运行 no-mistakes 启动交互式 TUI。不需要先提交——内置 wizard 处理分支创建、提交和推送。

no-mistakes -y 全自动模式，跳过交互，一键门禁。

适用于：想一站式完成"提交→审查→推送"的工作流。
直接从 TUI 中处理 findings：auto-fix 自动应用，ask-user 等你审批。
方式三：/no-mistakes（Agent Skill）
在 Claude Code 等 AI 编程助手中直接使用 /no-mistakes 命令。

你可以说："帮我把这个功能做完，然后 /no-mistakes"——Agent 会先完成任务，再自动触发门禁流水线。

也可以直接输入 /no-mistakes 对已完成的代码运行门禁。

底层使用 no-mistakes axi——非交互式 TOON 接口，适配 Agent 对话流程。
04三种触发方式总结
三种触发方式覆盖了不同场景：git push 适合已提交代码一键过门禁，TUI 适合交互式全流程控制，Agent Skill 让 AI 编程助手自动承接审查。选择最适合你工作流的即可。
