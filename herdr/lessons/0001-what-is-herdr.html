你开了三个 Claude Code 会话重构后端，又跑了两个 Codex 写前端，还有一个 Cursor 在那读代码。你的终端长什么样？
大概率是——开了 N 个终端标签页，每个标签页里跑着一个 Agent，翻来翻去找到底哪个跑完了、哪个报错了、哪个卡半天了。偶尔关掉电脑回家，回来发现某个会话中超时断开了，又得重新来一遍。
这问题说白了就是——tmux 太老，GUI 又太重。tmux 不支持 Agent 状态感知，你没法一眼看出哪个 pane 里的 Agent 卡了。各种 GUI Agent 管理器呢？大部分是 Mac-only 的闭源应用，把终端塞进一个仿冒窗口里，全屏 TUI 都渲染不对，更别提 SSH 连到服务器上了。

01herdr 到底干了件什么事？
你用过 tmux 吧？Session 持久化、分屏 pane、分离/重连——这些好东西 tmux 都有。但它有一个时代局限性：它诞生在 AI Agent 出现之前。tmux 不知道哪个 pane 里跑的是 Agent，不知道 Agent 是卡住了还是在工作，不知道哪个跑完了。
herdr 就是在 tmux 的这些功能基础上，长出了专门为 AI Agent 设计的新东西：
🔴
Agent 状态面板——一眼看清所有 Agent 的状态
侧边栏自动把每个 Agent 标为 🔴 阻塞 / 🟡 工作中 / 🔵 已完成 / 🟢 空闲。零配置，不需要手动写钩子。Claude Code、Codex、Cursor、GitHub Copilot 等 16+ Agent 开箱即识别。
📟
真正的终端，不是仿冒品
每个 Agent 获得一个真实的终端视图，全屏 TUI 也能完美渲染。不像那些 GUI 管理器把终端塞进一个 app 窗口里。
💤
断线不丢进度，去哪都能连
后台服务独立运行，关掉笔记本 Agent 继续跑。到家换台电脑 SSH 连回去，或者手机 SSH 上去看一眼进度——一切完好。
🗂️
Workspace / Tab / Pane 随心布局
按项目或目录组织工作区、标签、分屏。原生支持鼠标拖拽和点击，不用记快捷键也能上手。
🖥️
在哪都能跑
单个 ~10MB Rust 二进制，Linux 和 macOS 原生支持（Windows beta），零依赖。本机跑、服务器跑、SSH 远程连接，全都可以。
🔌
Agent 也能操作 herdr
本地 Unix Socket 暴露 API，Agent 可以自己创建 workspace、分屏、启动子 Agent、读取输出、订阅状态变更。这套机制让 Agent 之间互相协作成为可能。
02跟 tmux 和 GUI 管理器比，到底强在哪？
项目 README 里有一张对比表，我翻译一下核心结论：
能力 tmux GUI 管理器 herdr
持久化 Session✅—✅
断开/重连✅—✅
SSH 远程 + 任何终端运行✅—✅
Pane/Tab/Workspace✅✅✅
Agent 状态感知—✅✅
真正活在终端里✅—✅
真实终端视图✅—✅
原生鼠标支持—✅✅
轻量单二进制✅—✅
Agent 可编程操控❓❓✅
tmux 的持久化和 pane 功能确实强，但它不识别 Agent。你可以自己写钩子靠 bell 字符来判断，但每个 Agent 都得单独配，而且没有统一的全局面板来鸟瞰所有 Agent 的状态。GUI 管理器（像 Conductor、cmux、emdash）能看 Agent 状态，但它们本质上是 app——Mac-only、闭源，把终端画在窗口里而不是真正的终端。herdr 把两者的优点结合了：tmux 的轻量和持久化 + Agent 感知。
💡 一句话总结
tmux 给你持久化但是不认 Agent，GUI 管理器认 Agent 但是不够终端原生。herdr = tmux 的底子 + Agent 感知的大脑。
03安装和快速上手
安装就一条命令，支持 macOS、Linux 和 Windows（beta）：
# macOS / Linux
curl -fsSL https://herdr.dev/install.sh | sh

# Windows PowerShell（预览）
powershell -ExecutionPolicy Bypass -c "irm https://herdr.dev/install.ps1 | iex"
也可以用 Homebrew 或 mise 安装：
brew install herdr
# 或者
mise use -g herdr
装完直接跑：
herdr
它会启动或连接到一个后台服务，然后打开一个 workspace。一个 pane 里跑一个 Agent，鼠标可以直接拖拽分隔线、点击标签、分屏 p>
⌨️
关键快捷键
ctrl+b 是前缀键，按完松开再按动作键：
c — 新建标签 · v 或 - — 分屏 · shift+n — 新建 workspace
w — 切换 workspace · q — 断开（Agent 继续跑）
? — 查看全部快捷键
⚠️ 新手提示
• herdr 只留了一个保留键 ctrl+b，不影响你终端里其他快捷键的使用
• 断开后 Agent 不会停——herdr q 断开，用 herdr 重新连回去
• 想从本地连远程服务器上的 herdr：herdr --remote workbox 或 herdr --remote ssh://you@server:2222
• 图片粘贴等功能在远程模式下仍然可用，不会像纯 SSH+tmux 那样断掉
04远程模式——最让我眼前一亮的功能
herdr 的远程模式解决了一个真实痛点——你在本地 SSH 到远程服务器跑 tmux，图片粘贴、文件拖拽这些就废了。herdr 的 --remote 模式下，本地终端是客户端，远程 herdr 是服务端，图片粘贴这类操作走的是 herdr 协议而不是 SSH，所以功能不受影响。
# VPS 上跑 herdr，本地当客户端
herdr --remote ssh://you@yourserver:2222
搭配持久化 session，你在家合上电脑，到公司 SSH 连上服务器，herdr 重连，一切照旧。
05Agent 也能用 herdr——最有想象力的部分
herdr 暴露了一个本地 Unix Socket API。意思是——Agent 可以自己调用 herdr 的 API 来创建 workspace、分屏、启动子 Agent、读输出。
装一个 Agent 的官方集成：
herdr integration install claude
herdr integration install codex
herdr integration install copilot
这些集成让 Agent 感知 herdr 的 session 状态，甚至能在需要时自己分屏、创建新 workspace。想把 herdr 能力直接暴露给 Claude Code，还能装官方 skill：
npx skills add ogulcancelik/herdr --skill herdr -g
目前支持状态识别的 Agent 列表：Claude Code、Codex、Cursor、GitHub Copilot、Devin CLI、Droid、Amp、OpenCode、Grok CLI 等 16+。完整清单和服务端集成说明可以看 官方文档。
